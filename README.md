# The Non-Stationary Landauer Efficiency — open materials

*([Русская версия](./README.ru.md))*

Open-access materials for the article **«The Non-Stationary Landauer Efficiency: Memory Growth, Informational Nostalgia, and the Thermodynamics of Learning Systems»** (Russian primary; English translation included).

This is the companion to *«Landauer Efficiency of Self-Modeling: An Operational Scale of Vitality»* ([andriishin/landauer-self-modeling-oa](https://github.com/andriishin/landauer-self-modeling-oa)), extending the Landauer efficiency

```
η_L = I_pred / N_max,   N_max = E_actual / (k_B T ln 2)
```

from the stationary regime to a **non-stationary** environment with growing memory. A system holds its model against an environment that drifts faster than the model is refreshed, so a growing fraction of the stored bits no longer predicts anything — *informational nostalgia*. Two results are proved: **Lemma 1** (holding a bit against thermodynamic erosion costs strictly positive power, with an explicit lower bound) and **Lemma 2** (under slow Ornstein–Uhlenbeck drift and any finite refresh rate, the nostalgic fraction asymptotically stays above an explicit positive constant). Consequences: efficiency vanishes above a critical nostalgia, and an optimal rate of complete memory reset exists.

## Recompiling the PDFs

Precompiled PDFs are included for direct reading. To rebuild from the `.tex` you need a TeX distribution with `pdflatex` and `bibtex` (TeX Live or MiKTeX). The Springer Nature template files (`sn-jnl.cls`, `sn-*.bst`) are bundled, so no further downloads are required. The reference style is `sn-mathphys-ay` (author–year). Standard cycle for any document, e.g. `main`:

```bash
cd paper
pdflatex main.tex
bibtex   main
pdflatex main.tex
pdflatex main.tex
```

(replace `main` with `supplementary`, `main.ru`, or `supplementary.ru` for the other documents). The figures in `paper/figs/` are required for compilation and are regenerated by the simulations.

## Reproducing the simulations

Each `simulations/<experiment>/` is self-contained:

```bash
cd simulations/markov_drift_ou && pip install -r requirements.txt && python main.py
```

Seeded with `SEED=20260524` and per-run offsets; on the same NumPy version each run reproduces the figures and the numbers in `results_summary.{txt,json}`. Each folder has a `README.md` (English) and `README.ru.md` (Russian) documenting the model, parameters, and the result it establishes.
