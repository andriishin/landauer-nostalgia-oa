# iter-006 Шаг 1: OU + I_infinity (BNT-избыточность) — критический эксперимент

Дополнение к `../markov_drift_ou/` (iter-005). Та симуляция показала, что
асимптотика гипотезы S8.1 / (S8.2) Supplementary *не выполняется* с одношаговым MI-определением
`I_pred`: эмпирический наклон составил -0.089 против теоретических 28
(отношение -0.003, ошибочны и знак, и величина).

Эта директория тестирует **структурную поправку**: переопределить `I_pred` как
BNT-стилевую *кумулятивную* предсказательную информацию (excess entropy /
байесовскую избыточность), а не одношаговое MI, и заново проверить наклон.

## Две метрики

### Метрика A — MI с растущим окном (контроль)

`I^A_pred(t, T_w) = I(X_{t-T_w+1..t} ; X_{t+1..t+T_w})`

Для марковской цепи это *аналитически* равно `T_w * I_1 + O(1)`, где `I_1` —
одношаговое MI при текущей `P(t)`. Мы вычисляем его в замкнутой форме
(совместное распределение по `8^T_w` символам неразрешимо для `T_w > 3`). Это
**контроль**, делающий явным одношаговый потолок: `I^A_pred ≤ T_w ln k`, так
что информация на символ всё ещё ограничена сверху `ln k`.

### Метрика B — BNT-избыточность (основной тест)

Кумулятивная избыточная длина кода

```
L_excess(t) := sum_{s=1..t} [ log P_s(X_s | X_{s-1}) − log hat_P_s(X_s | X_{s-1}) ]
```

Это повыборочный аналог кумулятивного `KL(P_s || hat_P_s)`. Классическая теорема
о байесовской избыточности (BNT 2001, Class II / регулярная K-мерная модель)
предсказывает в стационарном режиме

```
E[L_excess(t)]  ~  (K/2) ln t  +  O(1).
```

В медленно дрейфующем OU-режиме `t << 1/sigma^2` то же масштабирование
переносится с заменой `t → lambda*t`, давая наклон **K/2 = 28** в `L_excess(t)`
по отношению к `ln(lambda t)`.

Мы также приводим *операциональную* форму

```
G_obs(t)   := sum_{s=1..t} [ log hat_P_s(X_s | X_{s-1}) + log k ]
eta^inf(t) := G_obs(t) / (R t)
```

так что операциональный тест наклона — это `eta^inf(t)*t` по отношению к
`ln(lambda t)`, теоретический наклон **K/(2R) = 28**. При конечном `sigma`
OU-дрейф добавляет *линейный по t* KL-канал поверх логарифмического члена BNT, а
линейный по `t` член нельзя поглотить в свободный член подгонки против
`ln(lambda t)` — он искажает наивный наклон по одному логарифму. Поэтому
коэффициент BNT снимается из **совместной подгонки**
`L_excess(t) = A*t + B*ln(lambda t) + C`, где с `K/2` сравнивается именно `B`.

## Параметры

| символ | значение | обоснование |
|---|---|---|
| k | 8 | как в статье |
| K = k(k-1) | 56 | свободные OU-логиты |
| lambda | 1e-3 | скорость возврата к среднему OU (статья §5.2) |
| sigma | **0.01** | **меньше, чем 0.1 в iter-005**: удерживает T<<1/sigma^2 на всей симуляции |
| T | 1e4 | = 1/sigma^2, ровно граница переходного процесса |
| n_runs | 30 | малое по спецификации (достаточно для наклона порядка O(10)) |
| theta0_std | 0.1 | начальный разброс логитов |
| measure_every | 100 | прореживание |
| R | 1 | скорость бюджета (безразмерная) |
| learn_c | 3.0 | начальный шаг Роббинса–Монро |
| окно подгонки | [1e3, 1e4] | t >> tau_E и t <= 1/sigma^2 |

## Что проверяет скрипт

Тест — это коэффициент BNT `B` из совместной подгонки
`L_excess(t) = A*t + B*ln(lambda t) + C`, сравниваемый с предсказанием
`K/2 = 28`. Наивный наклон `L_excess(t)` по одному `ln(lambda t)` и
операциональный наклон `eta^inf(t)*t` тоже приводятся, но при конечном `sigma`
оба доминируются линейным каналом дрейфа и *не* сравнимы напрямую с `K/2`.
Отношение `B/(K/2)` измеряет, насколько точно кумулятивная избыточная длина кода
следует логарифмическому масштабированию BNT Class II после отделения линейного
канала.

При `sigma = 0.01` (`results_summary.txt`) коэффициент `B` совместной подгонки
равен `9.5` (`B/(K/2) = 0.34`) — заметно ниже `K/2`. Наивный наклон по одному
логарифму при этом равен `87.8` (отношение `3.1`, *выше* `K/2`), а операциональный
наклон — `-8.7`; оба доминируются подогнанным линейным членом `A ~ 1.85e-2`, и
именно поэтому нужна совместная подгонка. Диагноз: OU-дрейф добавляет
*структурную* линейную по t KL-потерю, которая маскирует логарифмический член BNT
при конечном `sigma`. Адиабатический скан в `../markov_drift_ou_iinf_adiab/`
разделяет эти два канала и показывает, что `B/(K/2)` растёт к `1` при
`sigma^2/lambda -> 0`.
