# ML paradigm case: Permuted MNIST (§ 8.4)

## Что демонстрирует

Paradigm-case ML-суррогат для falsifiable-предсказания § 8.4 `paper/main.ru.md`
(ML-инстанциация Леммы 2), проверенный **с явным контролем выученности**.
Проверяемое предсказание: *истинная* ностальгическая фракция `nu^op_true` —
юниты, удерживающие устаревшую, когда-то предсказательную структуру, — должна
расти с частотой смены перестановки `rho_p` (то есть падать с каноническим
параметром `c = eta/rho_p`).

**Суррогат даёт честный отрицательный/обратный результат по наивному
прочтению** (см. «Честная рамка»): при контроле выученности монотонный рост
`nu` с `rho_p`, который дала бы наивная ablation-метрика, оказывается артефактом,
а суррогат демонстрирует иной, отчётливо определённый эффект.

Сканируемый канонический параметр:

```
c := M_refresh_dot * tau_E / |M_{<=t}|
для Permuted-задачи:  tau_E = 1/rho_p,  M_refresh_dot ~ eta*N,  |M| ~ N
=>  c ~ eta / rho_p
```

## Почему наивная ablation-метрика — квази-тавтология

Напрашивающаяся, но негодная операционализация: `nu^op` = *доля скрытых юнитов,
чьё зануление не повышает loss на текущей задаче выше малого порога*. Она
тавтологична. Если дрейф достаточно быстр, чтобы сеть не успевала выучить текущую
перестановку (accuracy близка к случайной — для 10 классов ≈ 0.1), то терять
нечего: зануление любого юнита не может повысить loss, и `nu^op -> 1`
**механически**. Такая метрика измеряет `(1 − выученность)`, а не «долю юнитов,
кодирующих устаревшую структуру», и её рост с `rho_p` подтверждал бы лишь
банальность *быстрый дрейф → сеть не обучается*, а не Лемму 2. Именно чтобы этого
избежать, ниже вводятся accuracy-gate и контроль прошлого.

## Честная метрика

Метрика опирается на два контроля.

1. **Accuracy-gate / dwell.** Ностальгия — это память, которая *удерживается* и
   *когда-то была предсказательной*; она осмысленна только когда текущая задача
   реально выучена. Время пребывания `dwell = 1/rho_p` выбрано `>>` числа шагов
   до сходимости на одной перестановке (при `eta = 0.1` сходимость ~1000 шагов),
   поэтому в основной части скана задача выучена (accuracy `>>` случайной). Точки
   с `accuracy < ACC_GATE = 0.80` помечаются **undertrained** и **исключаются**
   из любого вывода о `nu`.

2. **Истинная ностальгия через контроль прошлого.** Юнит **ностальгичен**, если
   его зануление (а) **не** повышает loss на текущей перестановке выше порога
   (бесполезен сейчас) **и** (б) **повышает** loss хотя бы на одной **недавней
   прошлой** перестановке (кодировал полезную тогда структуру). Храним
   тест-батчи `n_past = 3` свежайших прошлых перестановок.
   - `nu^op_true = mean[ (dL_тек ≤ thr) & (max_k dL_прошл_k > thr) ]`;
   - `dead_frac = mean[ (dL_тек ≤ thr) & (max_k dL_прошл_k ≤ thr) ]`
     (мёртвые/избыточные — нигде не полезны);
   - `useful_frac = mean[ dL_тек > thr ]`.

## Постановка

- **Датасет:** `sklearn.datasets.load_digits()` — реальные рукописные цифры 8×8
  (64 пикселя, 10 классов, 1797 образцов), поставляются вместе с sklearn, **без
  скачивания из сети**. Честный мини-аналог Permuted MNIST. Фолбэк —
  синтетический teacher-датасет, если sklearn недоступен.
- **Модель:** MLP `64 -> H -> 10` в чистом numpy (tanh-скрытый слой, softmax +
  cross-entropy), online SGD с learning rate `eta`. `H = 32` (достаточно крупно,
  чтобы у обученной сети был невырожденный сигнал stale-юнитов; при `H = 16` у
  выученной сети ностальгических юнитов практически нет во всём learned-диапазоне).
- **Дрейф:** глобальная перестановка координат `pi_t` переставляет пиксели;
  метки неизменны. На каждом шаге с вероятностью `rho_p` перестановка заменяется
  новой случайной; вытесненная перестановка попадает в очередь недавних прошлых.

## Параметры

| параметр | значение | смысл |
|---|---|---|
| `eta` | `0.1` | learning rate SGD (числитель `c`); крупнее, чтобы сходимость укладывалась в dwell |
| `H` | `32` | скрытых юнитов (единица ablation) |
| `N_STEPS` | `120 000` | SGD-шагов на прогон |
| `abl_threshold` | `0.05` | порог `dL` (бесполезен/полезен) |
| `n_past` | `3` | недавних прошлых перестановок для контроля прошлого |
| `ACC_GATE` | `0.80` | ниже — точка *undertrained*, исключается из вывода о `nu` |
| `N_runs` | `4` | усреднение по зёрнам |
| `rho_p` grid | `{3e-3, 1e-3, 5e-4, 3e-4, 1e-4}` | `dwell = 1/rho_p` от 333 до 10 000 |
| `SEED` | `20260524` | фиксированное зерно (байт-воспроизводимость) |

- Спирмен(`rho_p`, `nu^op_true`) = **−1.000** — истинная ностальгия **падает** с

- Спирмен(`c`, `nu^op_true`) = +1.000.
- Спирмен(accuracy, `nu^op_true`) = +1.000 — тривиальной антикорреляции с

  корреляция имеет противоположный знак и другую причину (ниже).

## Честная рамка

**Суррогат не подтверждает наивное предсказание «быстрее дрейф →
больше ностальгии». При контроле выученности этот эффект исчезает и
переворачивается.** Монотонный рост `nu` с `rho_p`, который дала бы наивная
ablation-метрика, — артефакт `(1 − выученность)`: его создаёт неспособность сети
обучиться при быстром дрейфе, а не удержание устаревшей структуры.

Что суррогат *действительно* показывает — реальный, отчётливо
определённый, но **иной** механизм:

- При **медленном дрейфе / долгом dwell** (`rho_p = 1e-4`, dwell = 10 000) сеть
  переспециализируется на каждую перестановку. После смены многие юниты, остро
  настроенные на *недавнюю прошлую* перестановку, мгновенно бесполезны на новой,
  но всё ещё детектируемо полезны на прошлой — ровно та сигнатура
  stale-retained, которую и ловит `nu^op_true` (`nu^op_true ≈ 0.40`).
- При **быстром (но всё ещё learned) дрейфе** (`rho_p = 1e-3`, dwell = 1 000)
  сеть консолидирует только широко-полезные юниты; permutation-специфичная
  структура почти не накапливается, и `nu^op_true ≈ 0.03`.

Итог: истинное направление здесь — *ностальгия растёт со **временем
переобучения на режим**, то есть с dwell `1/rho_p`*, а не с частотой смены. Это
информативно для § 8.4: суррогат изолирует подлинную stale-retained фракцию
(отделённую от never-useful/dead юнитов) и показывает, что в данной постановке
(малый MLP, online-SGD) доминирующий драйвер истинной ностальгии —
режим-специфичная переспециализация при долгом dwell, **а не** отставание памяти
от быстрого дрейфа. Наивное утверждение «монотонно с `rho_p`» не выдерживает
контроля выученности и не должно подаваться как подтверждение Леммы 2.

Это **order-of-magnitude суррогат механизма**, не прецизионная калибровка; малый
`load_digits` (8×8) MLP — намеренно минимальный зонд.

## Как запустить

```
cd simulations/permuted_mnist
python main.py
```

Повторный запуск даёт **байт-идентичные** числа (зерно `20260524`, прогоны
декоррелированы сдвигом `SEED + run`; wall-clock в сводках отсутствует).
