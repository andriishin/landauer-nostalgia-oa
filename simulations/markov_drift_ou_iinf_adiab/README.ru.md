# iter-006 Шаг 2: OU + I_infinity, адиабатический скан

## Цель

Шаг 1 (`../markov_drift_ou_iinf/`) показал, что в OU-дрейфовой марковской цепи
при `sigma=0.01, lambda=1e-3` (то есть `sigma^2/lambda = 0.1`) кумулятивная
избыточная длина кода `L_excess(t)` идеализированного онлайн-байесовского
обучающегося имеет правильную *функциональную* форму для предсказания BNT
Class II:

```
L_excess(t) = A * t + B * ln(lambda * t) + C        (совместная подгонка, R^2 = 0.9998)
```

но BNT лог-коэффициент вышел `B = 9.5`, всего ~34 % от теоретического
`K/2 = 28` (где `K = k(k-1) = 56`, `k = 8`).

Гипотеза состояла в том, что OU-дрейф добавляет *структурную* линейную по t
KL-потерю, которая хорошо отделена от BNT-избыточности *только* в адиабатическом
пределе `sigma^2/lambda -> 0`; при `sigma=0.01` вклад дрейфа доминирует над
лог-членом, занижая `B`.

Этот Шаг 2 сканирует `sigma in {0.01, 0.003, 0.001}` (то есть `sigma^2/lambda`
от `0.1` до `1e-3`, два порядка величины) и переподгоняет `B` на фиксированном
окне сравнения `[10^3, 10^4]` (то же окно, что и на Шаге 1) для каждого `sigma`.

## Результат

Подогнанный BNT-коэффициент `B` растёт **монотонно** к `K/2 = 28` по мере того,
как `sigma^2/lambda` уменьшается:

| sigma | sigma^2/lambda | T_total | B (окно сравнения) | B / (K/2) | R^2 |
|------:|---------------:|--------:|-------------------:|----------:|----:|
| 0.01  | 1.0e-1         | 50 000  | 15.15              | 0.541     | 0.9998 |
| 0.003 | 9.0e-3         | 50 000  | 21.85              | 0.780     | 0.9996 |
| 0.001 | 1.0e-3         | 200 000 | 23.84              | 0.851     | 0.9995 |

Итак, `B/(K/2)` растёт 0.54 -> 0.78 -> 0.85 по мере того, как система становится
более адиабатической, — это *качественная* сигнатура, предсказанная гипотезой.

(Число Шага 1 на том же окне было `B = 9.5`, `B/(K/2) = 0.34`. Слегка более
высокое значение 0.54 здесь при `sigma=0.01` согласуется с чуть большей
выборкой (`N=25` против `N=30` на Шаге 1) плюс другим зерном и векторизованным
сэмплером; тот же монотонный тренд сохраняется в любом случае.)

Оно *не* полностью замыкается при `sigma=0.001` — остаётся ещё ~15 % разрыв до
`K/2`. Две дополняющие диагностики в более широком скане подтверждают, что это
конечно-`sigma` поправка, а не другая асимптота:

  - R^2 совместной подгонки остаётся > 0.9995 на всём протяжении, так что
    анзац `A*t + B*ln(lambda t) + C` остаётся отличным описанием данных на
    `[10^3, 10^4]`.
  - «Доля потерь от дрейфа» `A * t / L_excess(t)` при `t = 10^4` падает с
    ~0.73 при `sigma = 0.01` до ~0.20 при `sigma = 0.001`, то есть линейный
    дрейфовый член выпадает, оставляя лог-член доминировать. См.
    `fig_adiab_Lexcess_scan.png` (правая панель).

## Оговорка о широком окне (отрицательный контроль)

Если расширить окно подгонки до `[10^3, min(T_total, 0.5/sigma^2)]`, `B`
продолжает расти за пределы `K/2` (столбец таблицы `B_wide` в
`results_summary.txt`):

  - sigma=0.001, широкое окно: `B = 57.0` (то есть `B/(K/2) = 2.0`).

Это превышение (overshoot) **не** является BNT-эффектом: при `t >> некоторого масштаба
перехода` двухчленный анзац `A*t + B*ln(lambda t)` исчерпывает ёмкость,
собственный шаг Роббинса–Монро `1/sqrt(t)` обучающегося оставляет остаточное
смещение `O(sqrt(t))`, и линейный `A`- и лог-`B`-каналы начинают поглощать
члены, которые им не должны принадлежать. Правильный ход — зафиксировать окно
на `[10^3, 10^4]` (окно Шага 1) для прямого сопоставления; именно на этом
столбце основано утверждение «только адиабатический».

## Что подтверждает скан

Адиабатически-асимптотическое утверждение `L_excess ~ (K/2) ln(lambda t)`
(гипотеза S8.1 / (S8.2) Supplementary) сохраняется как результат лишь в пределе
`sigma^2/lambda -> 0` (при окне подгонки, удерживаемом внутри переходного
процесса `t << 1/sigma^2`): эмпирический BNT-коэффициент сходится к
теоретическому значению `K/2`, с конечно-`sigma` поправкой порядка
`O(sigma^2/lambda)`. Эмпирическое свидетельство — монотонное приближение
`0.54 -> 0.78 -> 0.85` на скане в два порядка величины по `sigma^2/lambda`.
Наклон действительно движется к предсказанному `K/2` по мере приближения к
режиму, как качественно и должна делать асимптотика BNT-типа.

## Файлы

  - `model.py`           -- векторизованные пакетные OU + онлайн-ML обучающийся.
                            Все `N_runs` реализаций прогоняются в параллельных
                            массивах формы `(N, k, k)`, поэтому прогон
                            `T = 2*10^5` при `N = 25` завершается за ~34 с, а не
                            за часы.
  - `main.py`            -- запускает скан, подгонки, графики, пишет файлы сводки.
  - `results_summary.txt` / `.json`  -- машино- и человекочитаемые таблицы результатов.
  - `run.log`            -- копия stdout прогона.
  - `fig_adiab_B_vs_sigma2_lambda.png`  -- основной график, `B` vs `sigma^2/lambda`
                                           с пунктирной линией `K/2 = 28`.
  - `fig_adiab_Lexcess_scan.png`        -- (слева) наложенные кривые `L_excess(t)`
                                           с совместными подгонками; (справа)
                                           доля потерь от дрейфа `A*t/L_excess(t)`,
                                           сжимающаяся по мере перехода к адиабатике.

## Как воспроизвести

```
cd simulations/markov_drift_ou_iinf_adiab
python main.py
```

Полное время прогона: ~1 минута на современном ноутбуке. Сид = `20260524` на
каждый `sigma` (сдвинут на `int(1000*sigma)` для декорреляции зёрен между
точками скана).
