# Симуляции марковского дрейфа (§ 6.1 статьи *Нестационарная ландауэровская эффективность*)

Численная иллюстрация трёх режимов $\eta_L(t)$ и ностальгической доли $\nu(t)$
для PSP-суррогата (piecewise-stationary с пуассоновскими переключениями)
как аппроксимации OU-дрейфового класса (§ 5.2).

## Модель в одном абзаце

- Конечнозначная марковская цепь на $\Omega=\{1,\dots,k\}$ с $k=8$.
- Матрица переходов $P(t)$ кусочно-постоянна на интервалах
  $[T_i,T_{i+1})$; $\{T_i\}$ — пуассоновский процесс интенсивности $\lambda$;
  в каждый момент $T_i$ новая строчно-стохастическая матрица сэмплируется из
  Dirichlet$(\alpha)$ по каждой строке (мы используем $\alpha=0.3$, см. ниже).
- Наблюдатель удерживает $\hat P(t)$ — оценку матрицы переходов методом
  скользящего окна с лапласовским сглаживанием за последние $\tau_w$ шагов; для
  режима «без сброса» $\tau_w=\infty$.
- Предсказательная информация на горизонте $\tau$:
  $I_{\mathrm{pred}}(t,\tau) = I_{\mathrm{opt}}(P(t),\tau)
   - \mathbb{E}_{X_t\sim\pi(t)} D_{\mathrm{KL}}(P(t)^\tau(\cdot|X_t)\,\|\,\hat P(t)^\tau(\cdot|X_t))$,
  с обрезкой по нулю.
- $\nu(t)=1-I_{\mathrm{pred}}(t,\tau)/I_{\mathrm{opt}}(P(t),\tau)$.
- Размер памяти $|M_t|=k^2=64$ структурных параметров; используем стоимость
  $N_{\max}(t)=R\cdot t$ при $R=1$ (в единицах $k_B T\ln 2$ на шаг).
- $\eta_L(t)=I_{\mathrm{pred}}(t,\tau)/N_{\max}(t)$.

### Почему $\alpha=0.3$ и $\tau=1$?

Текст § 6.1 номинирует $\alpha=1$ (равномерное на симплексе). При $\alpha=1$
строки Dirichlet очень плоские, поэтому $\tau_{\mathrm{relax}}\sim 1.5$ шага и
$P^{\tau\ge 2}$ сходится к стационарному распределению в пределах численного
шума. Чтобы сравнение оставалось нетривиальным, мы сэмплируем из
Dirichlet$(\alpha=0.3)$, что даёт $\tau_{\mathrm{relax}}\sim 2{-}3$ шага (всё
ещё адиабатически при $\lambda=10^{-3}$,
$\lambda\tau_{\mathrm{relax}}\sim 3\times 10^{-3}\ll 1$), и приводим
$I_{\mathrm{pred}}$ на одношаговом горизонте $\tau=1$. Это методологический
выбор для суррогата, а не изменение аналитического класса § 5.2.

## Как запустить

```bash
pip install -r requirements.txt
python main.py
```

Скрипт выполняет ~250 отдельных симуляций длиной $10^4$–$10^5$ шагов каждая.
Полное время прогона на современном ноутбуке — примерно 5–15 минут.

- `../../paper/figs/fig3_nu_evolution.png` — $\nu(t)$ для режимов дрейфа без
  сброса и дрейфа со сбросом.
- `results_summary.txt` и `results_summary.json` — численные результаты,
  используемые в основном тексте § 6.1 (наклон подгонки $\eta_L\cdot t$ vs
  $\ln(\lambda t)$, $\nu_\infty$, оптимальное $\tau_w$, стационарное плато).

## Файлы

- `model.py` — случайные матрицы переходов, оценщик со скользящим окном,
  функционалы предсказательной информации, симулятор одиночного прогона.
- `main.py` — драйвер эксперимента, построение графиков, количественный анализ.
